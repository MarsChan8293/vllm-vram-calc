## 实现计划：合并模型选择输入框

### 目标
将现有的 4 个独立输入（3 个厂商选择框 + 1 个 HuggingFace 输入框）合并为一个统一的输入区域，包含：
- **厂商下拉框**（支持自定义输入）
- **模型下拉框**（根据厂商动态更新，支持自定义输入）
- **Fetch 按钮**（获取模型参数）

### 修改文件

1. **`app/lib/constants.ts`** - 添加厂商和模型预设数据
   ```typescript
   export const VENDOR_PRESETS = {
     deepseek: { name: 'DeepSeek', models: [...] },
     qwen: { name: 'Qwen', models: [...] },
     glm: { name: 'GLM', models: [...] },
     other: { name: '其他/自定义', models: [] },
   };
   ```

2. **`app/lib/types.ts`** - 添加模型选择相关类型
   ```typescript
   export interface ModelSelectorState {
     vendor: string;
     modelId: string;
   }
   ```

3. **`app/components/ModelSelector.tsx`** - 新建组件
   - 厂商下拉框（可编辑的 combobox）
   - 模型下拉框（可编辑的 combobox，根据厂商动态更新）
   - Fetch 按钮（复用现有的 HuggingFace 获取逻辑）

4. **`app/page.tsx`** - 使用新组件替换旧的选择框

### UI 设计
```
┌─────────────────────────────────────────────────────┐
│ 模型选择                                             │
├─────────────────────────────────────────────────────┤
│ 厂商: [DeepSeek ▼]    模型: [DeepSeek-V3.2 ▼]       │
│                       [Fetch]                        │
└─────────────────────────────────────────────────────┘
```

### 功能特性
- 选择厂商后，模型下拉框自动显示该厂商的模型列表
- 支持在输入框中直接输入自定义 Model ID
- 点击 Fetch 从 HuggingFace 获取模型参数并自动填充表单
